Portfolio Overview

  • Design
    • Architecture Diagram - The architecture diagram describes the objects present in the system, the replication of distributed objects, and the communication channels between objects.
    • Use Cases - The use case diagram describes the ways that agents in the system can interact with the elevator and shows relationships between these methods of interaction.
    • Scenarios and Sequence Diagrams - A list of use cases with their associated scenarios and sequence diagrams.
    • Requirements I - System Object Descriptions and Message Dictionary - A list of the sensors and actuators in the system and a message dictionary describing the messages that are required for the various controllers.
    • Requirements II - Distributed Controller Requirements - A list of detailed requirements for environmental objects and controllers in the system, including assumptions, interfaces, constraints, behaviors and state charts.
  • Traceability
    • Sequence Diagrams to Requirements Traceability - A traceability check from Sequence Diagrams to Requirements
    • Requirements to Contstraints Traceability - A traceability check from Requirements to Constraints
    • Statecharts to Code Traceability - Traceability table between the statecharts and code
